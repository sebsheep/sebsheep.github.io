Multi-target compilation

2014 October - December

Original idea: L. De Feo

  • Julia

See the Github repo.

This project aims to produce (cryptographic) code in several langages (namely C, Java and Python) which behaves exactly in the same manner.

The algorithms are specified in a "Box language": we plug wires on boxes and each boxe realizes some logic function. Then this language is translated into a "single assignment language", afterwards, the back end can compile into C, Java or Python.

Interactive proof assistant

Since 2015 December

Original idea: R. Sirdey

  • Python
  • Flask
  • AJAX
  • YAML

See a demonstration (in French).

The goal of this project is to illustrate to children that a mathematical proof is essentially "mechanic".

For that purpose, I've implemented a simple interactive proof assistant (like Coq) in Python. It 's an engine which can be queried with simple orders (as `Point A` to create a point named A). The orders are type checked before being executed, e.g. we can not say to the engine "assume A is parallel to d" if A is a point and d a line. Moreover, the error messages are designed to be as useful and comprehensible as possible.

This engine can be querried by a web application which emulates a console. This web app also adds an exercises management system.

The whole is highly configurable thanks to YAML config files : we can easily define our custom objects in the engine and exercises in the web app.

Find the sources (and more details in french) at : https://github.com/sebsheep/poussin.
