Outils d'aide à l'organisation de séjours pour Vitacolo

Septembre 2015 - Mai 2016

  • Python
  • Django
  • Twitter-bootstrap
  • jquery
  • jquery-ui

Vitanim est un outils d'aide à l'organisation de colonies vacances. Il a été commandé par Vitacolo, mais a vocation a être utilisé par n'importe quelle organisme de séjours de vacances.

Il gère :
  • un système de petites annonces,
  • un création hiérarchisée de séjour : un même lieu peut être partagé par des centres différents à différentes dates,
  • gestion des inscriptions d'enfants et de listes d'enfants,
  • recrutement d'animateurs avec des roles spécifiques sur les séjours (chaque ayant ses droits en lecture/écriture),
  • gestion des convois : affectation des animateurs sur les convois,
  • ...

De plus, il est possible d'alimenter la base de données grâce à des sources extérieures. Par exemple, Vitacolo utilise déjà cette fonctionnalité : toutes les inscriptions d'enfants enregistrées sur leur site commercial vitacolo.fr sont automatiquement synchronisées avec Vitanim.

Note : il existe un autre "Vitanim" remplissant à peu près les mêmes fonctions à l'adresse http://vitanim.fr . Ce site n'est pas le fruit de mon travail. Mon travail remplacera à terme ce site web.

Compilation multi cibles

Octobre - Décembre 2014

Idée originale : L. De Feo

  • Julia

Voir le repo Github.

Le but de ce projet est de produire du code (cryptographique) dans différents langages (ici C, Java et Python) qui se comporte exactement de la même manière.

Les algorithmes sont spécifiés dans un "langage de boîtes" : on branche des fils sur une boîte réalisant une certaine fonction logique. Ensuite, ce langage est converti vers un "single assignment language" puis le back end compile vers le C, Java ou Python.

Assistant de preuves interactif

Depuis décembre 2015

Idée originale : R. Sirdey

  • Python
  • Flask
  • AJAX
  • YAML

Voir une démo.

Le but de ce projet est d'illustrer aux enfants qu'une preuve mathématique est essentiellement un procédé "mécanique".

Pour cela, j'ai développé un assistant de preuves interactives simple (comme Coq) en Python. C'est un moteur qui peut recevoir des requêtes simples (comme "Point A" pour créer un point "A"). Une vérification de type sur les requêtes est effectuée avant qu'elles soient réellement exécutées : par exemple, on ne peut pas dire au moteur "A est parallèle à d" si A set un point et d est une droite. De plus, les messages d'erreurs sont construits de façon à être les plus compréhensibles et utiles possibles.

Ce moteur peut être interrogé via une application web qui émule une console. Cette application web met également un système de gestion d'exercices à disposition.

Le tout est hautement configurable grâce aux fichiers de configuration YAML : on peut facilement définir nos propres objets dans le moteur et exercices dans la web applications.

Plus de détails et les sources ici : https://github.com/sebsheep/poussin.
